## Step 1: Proto File

Place in `src/main/proto/computer_orders.proto`:

```protobuf
syntax = "proto3";

option java_package = "com.example.grpc.orders";
option java_outer_classname = "OrderProto";

message OrderMessage {
  enum Status {
    QUEUED = 0;
    PROCESSING = 1;
    COMPLETED = 2;
    FAILED = 3;
  }

  enum Equipment {
    KEYBOARD = 0;
    MOUSE = 1;
    WEBCAM = 2;
    MONITOR = 3;
  }

  string id = 1;
  string created_by = 2;
  Status status = 3;
  string created_at = 4;
  repeated Equipment equipment = 5;
}

message Empty {

}

message OrderMessageList {
  repeated OrderMessage orders = 1;
}

service OrderService {
  rpc Create(OrderMessage) returns (OrderMessage);
  rpc Get(Empty) returns (OrderMessageList);
}
```

**Key points:**
* `proto3` syntax is specified
* `OrderMessage` uses enums `Status` and `Equipment` for built-in validation
* `Empty` message handles `null` inputs (e.g., `Get` takes no meaningful argument)
* `OrderMessageList` wraps multiple `OrderMessage` objects for the `Get` response
* `java_package` places generated code in the correct Java package
* `java_outer_classname` sets the wrapper class to `OrderProto`

## Step 2: Generate gRPC Files

Run:

```bash
gradle build
```

This generates:
* `OrderProto.java` — contains `OrderMessage`, `Empty`, `OrderMessageList`, and their builders
* `OrderServiceGrpc.java` — contains `OrderServiceImplBase`, `OrderServiceBlockingStub`, etc.

Both are generated in `build/generated/source/proto/main/` and should **not be edited**.

## Step 3: Server Implementation

### `OrderServer.java`

```java
package com.example.grpc.orders;

import io.grpc.Server;
import io.grpc.ServerBuilder;
import io.grpc.stub.StreamObserver;

import java.io.IOException;
import java.util.logging.Logger;

public class OrderServer {

    private static final Logger logger = Logger.getLogger(OrderServer.class.getName());
    private static final int PORT = 5005;

    public static void main(String[] args) throws IOException, InterruptedException {
        Server server = ServerBuilder.forPort(PORT)
                .addService(new OrderServiceImpl())
                .build()
                .start();

        logger.info("gRPC Order Server started on port " + PORT);

        // Graceful shutdown hook
        Runtime.getRuntime().addShutdownHook(new Thread(() -> {
            logger.info("Shutting down gRPC server...");
            server.shutdown();
            logger.info("Server shut down.");
        }));

        server.awaitTermination();
    }

    /**
     * Implementation of OrderService defined in computer_orders.proto.
     * Equivalent to Python's OrderServicer class.
     */
    static class OrderServiceImpl extends OrderServiceGrpc.OrderServiceImplBase {

        /**
         * Create — accepts an OrderMessage and echoes it back.
         * Equivalent to Python's OrderServicer.Create.
         */
        @Override
        public void create(OrderProto.OrderMessage request,
                           StreamObserver<OrderProto.OrderMessage> responseObserver) {

            System.out.println("=== Order Created ===");
            System.out.println("  ID:         " + request.getId());
            System.out.println("  Created By: " + request.getCreatedBy());
            System.out.println("  Status:     " + request.getStatus());
            System.out.println("  Created At: " + request.getCreatedAt());
            System.out.println("  Equipment:  " + request.getEquipmentList());
            System.out.println("=====================");

            // Echo the same order back
            responseObserver.onNext(request);
            responseObserver.onCompleted();
        }

        /**
         * Get — returns a hardcoded list of orders.
         * Equivalent to Python's OrderServicer.Get.
         */
        @Override
        public void get(OrderProto.Empty request,
                        StreamObserver<OrderProto.OrderMessageList> responseObserver) {

            // Hardcoded order 1
            OrderProto.OrderMessage order1 = OrderProto.OrderMessage.newBuilder()
                    .setId("order-001")
                    .setCreatedBy("user-alice")
                    .setStatus(OrderProto.OrderMessage.Status.COMPLETED)
                    .setCreatedAt("2026-05-20T10:30:00Z")
                    .addEquipment(OrderProto.OrderMessage.Equipment.KEYBOARD)
                    .addEquipment(OrderProto.OrderMessage.Equipment.MONITOR)
                    .build();

            // Hardcoded order 2
            OrderProto.OrderMessage order2 = OrderProto.OrderMessage.newBuilder()
                    .setId("order-002")
                    .setCreatedBy("user-bob")
                    .setStatus(OrderProto.OrderMessage.Status.PROCESSING)
                    .setCreatedAt("2026-05-22T14:15:00Z")
                    .addEquipment(OrderProto.OrderMessage.Equipment.MOUSE)
                    .addEquipment(OrderProto.OrderMessage.Equipment.WEBCAM)
                    .addEquipment(OrderProto.OrderMessage.Equipment.KEYBOARD)
                    .build();

            // Bundle into OrderMessageList
            OrderProto.OrderMessageList orderList = OrderProto.OrderMessageList.newBuilder()
                    .addOrders(order1)
                    .addOrders(order2)
                    .build();

            System.out.println("=== Get Orders Called — Returning " 
                    + orderList.getOrdersCount() + " orders ===");

            responseObserver.onNext(orderList);
            responseObserver.onCompleted();
        }
    }
}
```

**Notes:**
* `OrderServiceImpl` extends `OrderServiceGrpc.OrderServiceImplBase` — the auto-generated base class
* Each RPC method receives a `request` and a `StreamObserver` for sending the response
* `responseObserver.onNext(message)` sends the response payload
* `responseObserver.onCompleted()` signals that the RPC is finished
* A shutdown hook is added for graceful cleanup

## Step 4: Client Implementation

### `OrderWriter.java` — Demonstrates `Create` RPC

```java
package com.example.grpc.orders;

import io.grpc.ManagedChannel;
import io.grpc.ManagedChannelBuilder;

import java.util.concurrent.TimeUnit;

public class OrderWriter {

    public static void main(String[] args) throws InterruptedException {
        // Connect to the gRPC server
        ManagedChannel channel = ManagedChannelBuilder
                .forAddress("localhost", 5005)
                .usePlaintext()
                .build();

        // Create a blocking (synchronous) stub
        OrderServiceGrpc.OrderServiceBlockingStub stub =
                OrderServiceGrpc.newBlockingStub(channel);

        // Build an OrderMessage using the Builder pattern
        OrderProto.OrderMessage order = OrderProto.OrderMessage.newBuilder()
                .setId("order-003")
                .setCreatedBy("user-charlie")
                .setStatus(OrderProto.OrderMessage.Status.QUEUED)
                .setCreatedAt("2026-05-24T20:00:00Z")
                .addEquipment(OrderProto.OrderMessage.Equipment.KEYBOARD)
                .addEquipment(OrderProto.OrderMessage.Equipment.MOUSE)
                .addEquipment(OrderProto.OrderMessage.Equipment.MONITOR)
                .build();

        System.out.println("Sending order to server...");

        // Call the Create RPC
        OrderProto.OrderMessage response = stub.create(order);

        System.out.println("Response received:");
        System.out.println("  ID:         " + response.getId());
        System.out.println("  Created By: " + response.getCreatedBy());
        System.out.println("  Status:     " + response.getStatus());
        System.out.println("  Created At: " + response.getCreatedAt());
        System.out.println("  Equipment:  " + response.getEquipmentList());

        // Cleanup
        channel.shutdown().awaitTermination(5, TimeUnit.SECONDS);
    }
}
```

### `OrderGetter.java` — Demonstrates `Get` RPC

```java
package com.example.grpc.orders;

import io.grpc.ManagedChannel;
import io.grpc.ManagedChannelBuilder;

import java.util.concurrent.TimeUnit;

public class OrderGetter {

    public static void main(String[] args) throws InterruptedException {
        // Connect to the gRPC server
        ManagedChannel channel = ManagedChannelBuilder
                .forAddress("localhost", 5005)
                .usePlaintext()
                .build();

        // Create a blocking stub
        OrderServiceGrpc.OrderServiceBlockingStub stub =
                OrderServiceGrpc.newBlockingStub(channel);

        // Call Get with an Empty message
        OrderProto.OrderMessageList response =
                stub.get(OrderProto.Empty.getDefaultInstance());

        System.out.println("Retrieved " + response.getOrdersCount() + " orders:\n");

        for (OrderProto.OrderMessage order : response.getOrdersList()) {
            System.out.println("--- Order ---");
            System.out.println("  ID:         " + order.getId());
            System.out.println("  Created By: " + order.getCreatedBy());
            System.out.println("  Status:     " + order.getStatus());
            System.out.println("  Created At: " + order.getCreatedAt());
            System.out.println("  Equipment:  " + order.getEquipmentList());
            System.out.println();
        }

        // Cleanup
        channel.shutdown().awaitTermination(5, TimeUnit.SECONDS);
    }
}
```

## Summary — Python vs Java Comparison

| Aspect | Python | Java |
|--------|--------|------|
| **Server file** | `main.py` | `OrderServer.java` |
| **Writer client** | `writer.py` | `OrderWriter.java` |
| **Getter client** | `getter.py` | `OrderGetter.java` |
| **Proto → code** | `grpc_tools.protoc` CLI | Gradle/Maven plugin (automatic) |
| **Message creation** | `OrderMessage(id="...", ...)` | `OrderMessage.newBuilder().setId("...").build()` |
| **Enum access** | `order_pb2.OrderMessage.QUEUED` | `OrderProto.OrderMessage.Status.QUEUED` |
| **Empty message** | `order_pb2.Empty()` | `OrderProto.Empty.getDefaultInstance()` |
| **List iteration** | `for order in response.orders` | `for (OrderMessage order : response.getOrdersList())` |
| **Type checking** | Runtime error | Compile-time error ✅ |
| **Boilerplate** | Less | More (but more explicit) |

---

## Key Takeaways

1. **Java's type system + protobuf** = errors caught at **compile time**, not runtime
2. The **Builder pattern** is the Java way to construct immutable protobuf messages
3. **Gradle/Maven plugins** automate code generation — no manual `protoc` commands needed
4. The overall architecture (server, client, proto) is **identical** between Python and Java — only the syntax differs
5. Java provides three stub types: **Blocking** (sync), **Future** (async with Future), and **Async** (callback-based) — giving more control over concurrency
