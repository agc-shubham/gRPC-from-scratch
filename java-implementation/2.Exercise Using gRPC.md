### Applying Using gRPC — Java

Using the following `proto` file that defines computer orders, set up a gRPC server **in Java** that accepts `OrderMessage`'s:

```protobuf
syntax = "proto3";

option java_package = "com.example.grpc.orders";
option java_outer_classname = "OrderProto";

message OrderMessage {
  enum Status {
    QUEUED = 0;
    PROCESSING = 1;
    COMPLETED = 2;
    FAILED = 3;
  }

  enum Equipment {
    KEYBOARD = 0;
    MOUSE = 1;
    WEBCAM = 2;
    MONITOR = 3;
  }

  string id = 1;
  string created_by = 2;
  Status status = 3;
  string created_at = 4;
  repeated Equipment equipment = 5;
}

message Empty {

}

message OrderMessageList {
  repeated OrderMessage orders = 1;
}

service OrderService {
  rpc Create(OrderMessage) returns (OrderMessage);
  rpc Get(Empty) returns (OrderMessageList);
}
```

---

### Task List

Set up and validate a gRPC server in Java that can handle `OrderMessage`:

- [ ] **Project setup** — Create a Gradle (or Maven) project with gRPC dependencies and the protobuf compiler plugin
- [ ] **Proto file** — Place `computer_orders.proto` in `src/main/proto/` with the `java_package` and `java_outer_classname` options
- [ ] **Generate code** — Run `gradle build` to generate Java classes from the proto file
- [ ] **Server** — Implement `OrderServiceImpl` extending `OrderServiceGrpc.OrderServiceImplBase`:
  - `Create` — accepts an `OrderMessage`, logs it, and returns it
  - `Get` — returns a hardcoded list of `OrderMessage` objects
- [ ] **Writer client** — Create a client that builds an `OrderMessage` and calls `Create`
- [ ] **Getter client** — Create a client that calls `Get` with an `Empty` message and prints the returned orders
- [ ] **Validate** — Run the server in one terminal and test with both clients in separate terminals

### Hints

1. Remember that Java uses the **Builder pattern** to construct protobuf messages:
   ```java
   OrderProto.OrderMessage order = OrderProto.OrderMessage.newBuilder()
           .setId("order-001")
           .setCreatedBy("user-42")
           .setStatus(OrderProto.OrderMessage.Status.QUEUED)
           .setCreatedAt("2026-05-24T12:00:00Z")
           .addEquipment(OrderProto.OrderMessage.Equipment.KEYBOARD)
           .addEquipment(OrderProto.OrderMessage.Equipment.MONITOR)
           .build();
   ```

2. For `repeated` fields, use `addEquipment()` (singular) to add one item at a time, or `addAllEquipment()` with a list.

3. The `Get` RPC takes an `Empty` message. In Java, you can create one with:
   ```java
   OrderProto.Empty empty = OrderProto.Empty.getDefaultInstance();
   ```

4. Your server should extend `OrderServiceGrpc.OrderServiceImplBase` and override both `create()` and `get()` methods.

5. Use `StreamObserver<T>` for sending responses:
   ```java
   responseObserver.onNext(responseMessage);
   responseObserver.onCompleted();
   ```
